# 阶段 5：端口转发

## 目标

按照 `docs/开发文档.md` 的 3.6 阶段要求，本阶段接入真实 SSH tunnel 端口转发能力：

- Local forwarding：本地监听端口转发到远端目标。
- Remote forwarding：远端监听端口转发到本地目标。
- Dynamic SOCKS forwarding：本地 SOCKS5 代理通过 SSH 访问远端可达目标。
- 端口转发规则保存、编辑、删除、启动、停止和状态事件。
- 本地监听端口占用返回 `FORWARD_PORT_IN_USE`。

## 后端实现

新增 `com.termira.forwarding.ForwardingManager`，负责转发规则运行态：

- `forward.list`
- `forward.create`
- `forward.update`
- `forward.delete`
- `forward.start`
- `forward.stop`
- `forward.statusChanged`

运行态状态：

- `starting`
- `running`
- `stopping`
- `stopped`
- `failed`

规则仍由 `ProfileStore` 持久化到 `forward_rules`，本阶段补充 `auto_start` 字段，并在初始化时自动兼容旧表结构。

SSHJ 能力映射：

- local：`SSHClient.newLocalPortForwarder`
- remote：`SSHClient.getRemotePortForwarder().bind`
- dynamic：Termira 内置 SOCKS5 握手和 relay，按请求创建 `DirectConnection`

## 前端实现

桌面端“转发”工具面板已从占位态改为真实操作界面：

- 新建和编辑 local、remote、dynamic 规则。
- 表单校验名称、监听地址、监听端口、目标地址和目标端口。
- 根据当前已连接终端 session 启动规则。
- 支持停止、删除、刷新列表。
- 订阅 `forward.statusChanged` 更新运行态。
- 失败规则展示错误信息。

## 验收步骤

### 本地端口转发

1. 连接一台 SSH 主机。
2. 新建 local 规则：
   - 监听地址：`127.0.0.1`
   - 监听端口：例如 `18080`
   - 目标地址：远端可访问地址，例如 `127.0.0.1`
   - 目标端口：远端服务端口，例如 `22` 或测试 HTTP 服务端口。
3. 启动规则。
4. 从本机访问 `127.0.0.1:18080`。
5. 停止规则后再次访问应失败，端口应释放。

### 远程端口转发

1. 确认 SSH 服务端允许 `AllowTcpForwarding`。
2. 新建 remote 规则：
   - 监听地址：通常为 `127.0.0.1`
   - 监听端口：远端空闲端口。
   - 目标地址：本机可访问地址。
   - 目标端口：本机服务端口。
3. 启动规则。
4. 从远端访问监听端口，应转发到本机目标服务。
5. 停止规则后远端端口释放。

### Dynamic SOCKS

1. 新建 dynamic 规则：
   - 监听地址：`127.0.0.1`
   - 监听端口：例如 `10808`
2. 启动规则。
3. 配置本机请求使用 SOCKS5 `127.0.0.1:10808`。
4. 访问远端可达目标。

### 端口占用

1. 本机占用某个端口。
2. 新建 local 或 dynamic 规则使用该端口。
3. 启动时应返回 `FORWARD_PORT_IN_USE`，UI 展示失败信息。

## 自动化验证

已补充 Java E2E 测试 `ForwardingManagerE2ETest`：

- local 转发到远端 `127.0.0.1:22` 并读取 SSH banner。
- local 端口占用返回 `FORWARD_PORT_IN_USE`。
- dynamic SOCKS 转发到远端 `127.0.0.1:22` 并读取 SSH banner。
- remote 转发从远端监听端口转回本机 one-shot 服务。

运行真实服务器验证：

```bash
read -s -r TERMIRA_E2E_SSH_PASSWORD
export TERMIRA_E2E_SSH_PASSWORD
TERMIRA_E2E_SSH_HOST=119.91.123.116 TERMIRA_E2E_SSH_USER=ubuntu mvn -Dmaven.repo.local=.m2/repository -f apps/backend-java/pom.xml -Dtest=ForwardingManagerE2ETest test
```

## 已知约束

- `auto_start` 已持久化，本阶段 UI 可配置；自动随连接启动的调度留给后续连接生命周期增强。
- Remote forwarding 依赖服务端 SSH 配置；服务端拒绝时返回 `FORWARD_OPERATION_FAILED`。
- Dynamic SOCKS 当前支持 SOCKS5 no-auth CONNECT。
