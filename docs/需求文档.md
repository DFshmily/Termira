# Termira 需求文档

## 1. 文档目标

本文档定义 Termira 第一阶段产品需求、第二阶段规划、功能边界、UI 风格原则和验收口径。Termira 是一个 clean-room 设计的 SSH/服务器管理客户端，只参考公开产品能力和通用行业交互，不使用 FinalShell 或 Termius 的源码、品牌、图标、专有文案、截图或像素级布局。

第一阶段目标是沉淀并实现一个 macOS 优先的完整 SSH 客户端，后续再扩展 Windows 客户端和更强的企业能力。

## 2. 产品定位

Termira 面向开发者、运维工程师、后端工程师和需要频繁管理服务器的技术用户。产品启动后直接进入工作台，不做营销首页，不用大面积装饰性视觉。

核心价值：

- 快速找到主机。
- 快速建立 SSH 连接。
- 快速打开终端。
- 快速管理远程文件。
- 快速配置和启停端口转发。
- 在同一个工作台内查看服务器状态、进程、网络和常用命令。

默认语言：

- 软件默认界面语言为中文。
- 保留英文语言资源，便于后续国际化。
- 语言切换入口放在设置中，不在工作台顶部长期占用主操作区域。

第一阶段产品形态：

- macOS 桌面客户端。
- Electron + React 前端。
- Java SSH 内核 sidecar。
- 本地配置与本地加密 Vault。
- 单机本地使用，不依赖云账号。

## 3. 参考边界

### 3.1 FinalShell 功能参考

Termira 第一阶段功能按 FinalShell 公开能力体系规划，包括：

- SSH 连接管理。
- 终端会话。
- SFTP 文件管理。
- 端口转发。
- 服务器监控。
- 进程管理。
- 快捷命令。
- 主机分组。
- 本地保存连接配置。

Termira 不复制 FinalShell：

- 不使用 FinalShell 源码。
- 不使用 FinalShell 图标、Logo、界面截图。
- 不复刻 FinalShell 像素级布局。
- 不复制 FinalShell 专有文案。
- 不复刻 FinalShell 本地凭据加密细节。

### 3.2 Termius UI 风格参考

Termira UI 方向参考成熟 SSH 客户端的克制专业感，整体更接近 Termius 这类现代工具的审美：

- 深色优先。
- 低噪音。
- 高对比。
- 信息密度适中。
- 图标精致。
- 状态反馈明确。
- 动画轻量自然。
- 界面直接进入工作台。

Termira 不复制 Termius：

- 不使用 Termius 品牌、Logo、图标或截图。
- 不复制 Termius 专有文案。
- 不进行像素级复刻。
- 不照搬其商业账户、团队空间和云同步表达。

## 4. 用户角色

### 4.1 个人开发者

需要管理少量服务器，频繁 SSH 登录、上传文件、查看日志、临时转发端口。

典型目标：

- 按名称搜索主机。
- 一键连接。
- 打开多个终端标签。
- 上传配置文件或构建产物。
- 临时开启本地端口转发访问远程服务。

### 4.2 运维工程师

需要管理多组服务器，关注连接效率、监控状态、进程和文件操作。

典型目标：

- 按项目、环境、区域分组主机。
- 同时打开多个会话。
- 查看 CPU、内存、磁盘、网络。
- 快速定位高负载进程。
- 使用常用命令模板减少重复输入。

### 4.3 后续企业用户

需要更强的安全、审计、跳板机、团队配置同步和权限管理。企业能力不进入第一阶段完整实现，但在架构中保留扩展位置。

## 5. 第一阶段功能范围

### 5.1 主机管理

必须支持：

- 新增、编辑、删除主机。
- 主机分组。
- 主机搜索。
- 收藏主机。
- 最近连接。
- 快速连接。
- 主机备注。
- 主机标签。
- 连接端口配置。
- 用户名配置。
- 认证方式配置。

主机列表展示字段：

- 主机名称。
- 地址。
- 用户名。
- 端口。
- 分组。
- 标签。
- 最近连接时间。
- 当前连接状态。

连接状态包括：

- 未连接。
- 连接中。
- 已连接。
- 断开。
- 认证失败。
- 网络不可达。
- 超时。

### 5.2 SSH 终端

必须支持：

- 交互式 SSH shell。
- 多标签终端。
- 同一主机多会话。
- 终端输入输出流式传输。
- 终端窗口 resize 同步到远端 PTY。
- 复制、粘贴。
- 清屏。
- 断开连接。
- 重连。
- 连接失败展示明确错误。

终端基本配置：

- 字体族。
- 字号。
- 行高。
- 光标样式。
- 主题颜色。
- 滚动缓冲行数。

第一阶段不要求：

- 终端分屏。
- 协作终端。
- 会话回放。
- 审计录像。

### 5.3 SFTP 文件管理

必须支持：

- 远程目录浏览。
- 返回上级目录。
- 刷新目录。
- 上传文件。
- 下载文件。
- 删除文件。
- 重命名。
- 新建目录。
- 文件大小展示。
- 修改时间展示。
- 权限展示。
- 文件传输队列。
- 传输进度。
- 传输失败重试。

推荐交互：

- 右侧工具区或独立面板展示远程文件。
- 文件操作使用工具栏图标和右键菜单。
- 上传/下载进度在底部任务区展示。

第一阶段不要求：

- 本地文件双栏完整文件管理器。
- 远程文件在线编辑。
- 断点续传。
- 批量权限修改高级面板。

### 5.4 端口转发

必须支持：

- 本地端口转发。
- 远程端口转发。
- 动态 SOCKS 转发。
- 端口规则新增、编辑、删除。
- 端口规则启停。
- 端口占用错误提示。
- 转发状态展示。

端口转发规则字段：

- 名称。
- 类型：local、remote、dynamic。
- 本地监听地址。
- 本地端口。
- 远程目标地址。
- 远程目标端口。
- 绑定的主机配置。
- 是否自动启动。
- 当前状态。

状态包括：

- 未启动。
- 启动中。
- 运行中。
- 停止中。
- 已停止。
- 失败。

### 5.5 服务器监控

必须支持通过 SSH 执行只读命令采集基础监控数据：

- CPU 使用率。
- 内存使用率。
- 磁盘使用率。
- 网络收发速率。
- 系统负载。
- 运行时间。

监控面板要求：

- 轻量刷新。
- 失败时展示不可用原因。
- 不阻塞终端输入输出。
- 不因为监控命令失败断开 SSH 主会话。

第一阶段可接受：

- Linux 优先。
- macOS 远端尽量兼容。
- Windows 远端不作为第一阶段监控重点。

### 5.6 进程管理

必须支持：

- 查看进程列表。
- 按 CPU、内存、PID、名称排序。
- 搜索进程。
- 刷新进程。
- 结束进程前二次确认。

第一阶段进程管理通过 SSH exec 命令实现，不引入远端 agent。

### 5.7 快捷命令

必须支持：

- 新增快捷命令。
- 编辑快捷命令。
- 删除快捷命令。
- 按分组管理快捷命令。
- 一键发送到当前终端。

快捷命令字段：

- 名称。
- 命令内容。
- 分组。
- 备注。

第一阶段不要求：

- 参数化命令表单。
- 团队共享命令库。
- 命令审批。

### 5.8 本地加密保存

必须支持：

- 保存主机配置。
- 保存密码。
- 保存私钥 passphrase。
- 可选保存私钥内容。
- 主密码可选。
- 不明文保存敏感凭据。

第一阶段不支持：

- macOS Keychain。
- Windows Credential Manager。
- 云同步。
- 团队 Vault。

### 5.9 已明确放到第二阶段的安全能力

以下能力第一阶段不实现：

- known_hosts。
- 首次连接指纹确认。
- 主机指纹变更阻断。
- macOS Keychain。
- Windows Credential Manager。

第一阶段连接未知主机时不阻断，安全风险必须在安全设计文档中明确。

## 6. 第二阶段功能规划

第二阶段包括：

- known_hosts。
- 首次连接指纹确认。
- 指纹变更阻断。
- macOS Keychain。
- Windows Credential Manager。
- 跳板机和 ProxyJump 增强。
- HTTP/SOCKS 代理配置增强。
- SSH agent 集成。
- 云同步和端到端加密。
- 配置导入导出。
- 批量命令。
- 审计日志。
- Windows 客户端。
- 可选 Telnet/RDP/VNC 等协议评估。

## 7. UI 与交互要求

### 7.1 整体风格

UI 必须满足：

- 深色优先。
- 现代 macOS 开发者工具风格。
- 克制、专业、高效。
- 不做营销感首页。
- 不做大面积渐变。
- 不做卡片堆叠式首页。
- 启动后直接进入工作台。

### 7.2 工作台布局

建议采用四区布局：

- 左侧导航：主机、分组、收藏、最近连接。
- 顶部标签：打开的终端会话和工具标签。
- 中央主区域：终端。
- 右侧或底部工具区：文件、转发、监控、进程、快捷命令。

布局要求：

- 用户能在首屏看到主机入口。
- 用户能在首屏看到终端区域。
- 工作台首屏不展示工程基线、后端调试面板或营销内容。
- 后端运行状态、语言切换等偏好和诊断能力统一放入设置。
- 用户能快速打开文件和端口转发面板。
- 多标签数量增加时布局不崩。
- 长主机名、长路径、长命令不溢出。

### 7.3 视觉细节

必须具备：

- 统一 spacing token。
- 统一 color token。
- 统一 typography token。
- 细腻边框。
- 清晰分割线。
- hover 状态。
- selected 状态。
- focus 状态。
- disabled 状态。
- loading 状态。
- error 状态。

### 7.4 图标

图标策略：

- 使用通用开源图标库，例如 lucide-react。
- 按功能选择熟悉符号。
- 不绘制或复制品牌图标。
- 图标按钮必须有 tooltip 或 aria-label。

## 8. 非功能需求

### 8.1 性能

- 应用冷启动目标小于 3 秒。
- 打开主窗口后 UI 不等待 Java sidecar 完成全部初始化。
- 终端输入输出延迟应尽量低。
- SFTP 大文件传输不阻塞 UI。
- 监控刷新不阻塞终端。

### 8.2 稳定性

- Java sidecar 崩溃时 Electron 应展示错误并允许重启。
- 单个 SSH 会话失败不影响其他会话。
- 单个 SFTP 任务失败不影响传输队列。
- 端口转发失败不影响终端会话。

### 8.3 可维护性

- 前端、Electron main、Java sidecar 分层清晰。
- IPC 协议版本化。
- 数据 schema 版本化。
- 错误码统一。
- 日志脱敏。

## 9. 验收标准

第一阶段验收必须满足：

- 应用启动后进入工作台。
- 应用默认显示中文界面，并可在设置中切换英文。
- 能创建主机配置。
- 能用密码连接 SSH。
- 能用私钥连接 SSH。
- 能打开交互终端。
- 能打开多个终端标签。
- 能执行 SFTP 目录浏览。
- 能上传和下载文件。
- 能配置并启动本地端口转发。
- 能配置并启动远程端口转发。
- 能配置并启动动态 SOCKS 转发。
- 能展示基础监控信息。
- 能查看进程列表。
- 能保存和读取本地加密凭据。
- UI 符合深色、克制、专业、高效的方向。
