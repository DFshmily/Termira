# 阶段 6：监控、进程与快捷命令

## 范围

本阶段按照 `docs/开发文档.md` 的 3.7 要求，实现通过 SSH exec 的服务器监控、进程管理和快捷命令能力，不安装远端 agent。

## 后端实现

- 新增 `MonitorManager`，通过独立 SSH exec 通道采集 Linux `/proc`、`df` 数据，默认 3 秒刷新，单次命令 2 秒超时。
- 新增 `monitor.start`、`monitor.stop`、`monitor.snapshot` IPC 方法。
- 监控采集项包含 CPU、内存、磁盘、网络收发速率、load average 和 uptime。
- 新增 `ProcessManager`，通过 `ps` 获取进程列表，通过受限信号白名单执行 `process.kill`。
- 新增 `process.list`、`process.kill` IPC 方法，列表刷新会发送 `process.listUpdated`。
- 复用现有 `quick_commands` 表，补齐 `command.list`、`command.create`、`command.update`、`command.delete`、`command.sendToTerminal`。
- `command.sendToTerminal` 会把命令写入当前终端 channel，并自动补换行。

## 前端实现

- 监控面板从占位状态升级为真实指标卡片和自动刷新状态。
- 进程面板支持刷新、搜索、按 PID/CPU/MEM/名称排序，以及 kill 前二次确认。
- 快捷命令面板支持新增、编辑、删除、分组展示、全局/当前主机绑定，以及一键发送到当前终端。
- 监控、进程和快捷命令均复用当前活动 SSH 会话，不阻塞终端输入输出。

## 验证记录

- `npm run typecheck -w apps/desktop`：通过。
- `npm run typecheck -w packages/shared`：通过。
- `npm run test -w apps/desktop`：通过。
- 真实服务器 E2E：通过 `MonitorProcessE2ETest` 连接测试服务器，完成监控采集、进程列表读取、启动临时 `sleep` 进程并通过 `process.kill` 结束。

## 注意事项

- 当前监控采集以 Linux 为优先目标；Windows 远端不在第一阶段范围内。
- 监控命令失败会返回不可用原因并展示在面板，不会断开 SSH 主会话。
- kill 只允许 `TERM`、`KILL`、`INT`、`HUP`，且拒绝 PID 小于等于 1 的请求。
