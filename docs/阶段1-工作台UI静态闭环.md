# 阶段 1：工作台 UI 静态闭环

## 目标

按照 `docs/开发文档.md` 的 3.2 要求，先完成 Termius 风格的工作台 UI 静态闭环。本阶段只使用 mock 数据，不接真实 SSH、SFTP、端口转发、监控采集或快捷命令执行。

## 已交付

- 左侧主机树：包含收藏、最近连接、全部主机和主机搜索。
- 主机状态：覆盖已连接、连接中、未连接、失败等静态状态。
- 中央终端：多标签终端壳、长标签截断、长路径输出、终端工具按钮。
- 右侧工具区：文件、转发、监控、进程、快捷命令入口首屏可见，并支持折叠为窄工具栏。
- 分栏工作区导航：最左侧固定应用栏承载主机、设置入口；主机列表作为独立栏位，可一键折叠。
- 终端标签栏：参考桌面 SSH 客户端的顶部会话标签，使用深色胶囊标签、图标、弱化关闭按钮和独立新增入口。
- 工具切换栏：取消顶部工具导航，统一收敛到右侧工具侧栏，避免终端顶部重复控件。
- 设置页：设置入口独立于主机列表，进入后展示独立设置侧栏和偏好内容区。
- 一体化运维视图：参考 FinalShell 的终端、SFTP、监控同屏工作方式，默认保留右侧 SFTP/工具面板。
- 终端标签闭环：所有标签均可关闭，关闭当前标签后自动切换到下一个可用标签。
- 文件面板：远程路径、文件列表、上传/下载工具栏、传输队列、失败状态。
- 端口转发面板：本地、远程、动态 SOCKS 规则和运行、启动中、失败、停止状态。
- 监控面板：CPU、内存、磁盘、网络 mock 指标，包含刷新中与局部错误状态。
- 进程面板：进程表、搜索、禁用结束按钮状态。
- 快捷命令面板：分组命令、发送按钮、禁用状态和长命令截断。
- 深色主题 token：颜色、间距、圆角、字体、边框、focus、hover、selected、disabled、loading、error 状态。
- Renderer 浏览器预览：静态工作台不依赖 Electron preload，可直接在 Vite 浏览器预览中验收布局与交互。

## 验收关注点

- 启动后直接进入工作台，不展示营销页或工程基线页。
- 首屏可见主机、终端、文件、端口转发、监控、进程和快捷命令入口。
- 长主机名、长路径、长命令、多终端标签不会撑破布局。
- 用户界面不展示 Java、后端进程或连接服务控制；连接引擎作为实现细节，排障能力留给开发日志或后续诊断入口。
- 真实连接能力仍留给后续阶段，不在本阶段伪装成可用能力。
