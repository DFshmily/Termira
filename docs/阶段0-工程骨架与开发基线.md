# 阶段 0：工程骨架与开发基线

日期：2026-04-29

## 目标

本阶段按 `docs/开发文档.md` 的 3.1 要求完成工程基线：

- 创建 monorepo。
- 建立 Electron + React + TypeScript 桌面前端。
- 建立 Maven Java sidecar。
- 打通 Electron main 启动 Java sidecar 的最小链路。
- 实现 `backend.ready`、`app.ping`、`app.shutdown`。
- Renderer 默认进入主机工作台；后端在线、Ping、离线状态收纳到设置的运行状态区域。
- 软件默认界面语言为中文，同时保留英文语言资源，语言切换入口位于设置。
- 建立基础日志目录。

## 交付内容

- `apps/desktop`：Vite React renderer、Electron main、preload。
- `apps/backend-java`：Maven Java sidecar、stdio JSON-RPC、fat jar 打包。
- `packages/shared`：前端共享 IPC 与领域类型。
- 根目录 workspace 脚本：`npm run dev`、`npm run test`、`npm run build`。
- 产品首屏为 clean-room 的 Termius 风格工作台布局：左侧主导航、主机列表、搜索、静态终端壳、连接信息面板。
- 设置页承载语言切换和 Java sidecar 运行状态，不在首屏暴露“工程基线”开发页面。

## IPC 行为

Java sidecar 使用 line-delimited JSON：

- 启动后输出 `backend.ready` event。
- 收到 `app.ping` 返回 `pong`、协议版本和后端版本。
- 收到 `app.getVersion` 返回协议版本和后端版本。
- 收到 `app.shutdown` 返回 accepted 并退出。
- 未知方法返回 `IPC_UNKNOWN_METHOD`。

Electron main 负责：

- 创建安全 BrowserWindow。
- 通过 preload 暴露 `window.termira.invoke/on/off`。
- 使用 stdio 管理 Java sidecar。
- 过滤非 JSON stdout，避免 Maven 包装器日志污染 IPC。
- 广播 `backend.ready`、`backend.exited`、`backend.error`。

## 开发命令

```bash
npm install
npm run dev
npm run test
npm run build
npm audit --json
```

Maven 依赖缓存使用仓库内 `.m2/repository`，避免写入全局 `~/.m2`：

```bash
mvn -Dmaven.repo.local=.m2/repository -f apps/backend-java/pom.xml test
mvn -Dmaven.repo.local=.m2/repository -f apps/backend-java/pom.xml clean package
```

## 当前边界

阶段 0 不实现真实 SSH、SFTP、端口转发和 Vault。当前工作台使用 mock 主机数据和静态终端壳，为后续阶段接入真实连接能力预留界面结构。

## 注意事项

- Maven shade 当前有 `module-info.class` 和 `META-INF` 重叠资源 warning；阶段 0 fat jar 可正常运行，后续引入更多 Java 依赖时再统一治理。
- Electron 开发模式通过 quiet Maven 启动 sidecar。若后续希望进一步提升启动速度，可在 `npm run dev` 前先执行 Java package，并让 Electron 直接使用已产出的 jar。
