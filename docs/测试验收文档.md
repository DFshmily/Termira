# Termira 测试验收文档

## 1. 文档目标

本文档定义 Termira 第一阶段测试策略、测试类型、真实服务器验收流程、UI 验收标准、安全验收边界和回归清单。

第一阶段测试重点：

- Java SSH 内核正确性。
- Electron 与 Java sidecar IPC 稳定性。
- 终端交互稳定性。
- SFTP 文件操作可靠性。
- 端口转发可用性。
- 本地 Vault 不明文保存敏感信息。
- UI 符合专业克制的工作台风格。

## 2. 测试环境

### 2.1 本地环境

建议：

- macOS 13+。
- Node.js 20+。
- JDK 17 或 21。
- Maven 3.9+。
- 可连接互联网安装依赖。

### 2.2 真实服务器

由用户提供非生产服务器。

服务器要求：

- Linux 优先。
- 开启 SSH。
- 提供测试账号。
- 测试账号允许 SFTP。
- 测试账号允许创建临时文件。
- 如需 remote forwarding，服务端需允许对应配置。

不得使用生产账号进行第一阶段验收。

## 3. 单元测试

### 3.1 前端单元测试

覆盖：

- 主机搜索过滤。
- 分组树展开折叠。
- 标签页状态切换。
- 端口规则表单校验。
- SFTP 文件大小格式化。
- 传输进度格式化。
- 错误码到用户文案映射。

### 3.2 Java 单元测试

覆盖：

- HostProfile 校验。
- ForwardRule 校验。
- Vault 加密写入。
- Vault 解密读取。
- 主密码正确解锁。
- 主密码错误拒绝。
- CredentialRef 查找。
- 错误码映射。
- 监控命令输出解析。
- 远程路径规范化。

### 3.3 IPC 单元测试

覆盖：

- request 解析。
- response 序列化。
- event 序列化。
- unknown method。
- invalid params。
- timeout。
- sidecar 退出。

## 4. 集成测试

### 4.1 SSH 密码登录

步骤：

1. 创建主机配置。
2. 选择 password 认证。
3. 输入用户名和密码。
4. 点击连接。
5. 打开终端。
6. 执行 `whoami`。

预期：

- 连接成功。
- 终端出现 shell prompt。
- `whoami` 输出当前测试账号。
- UI 状态为已连接。

### 4.2 SSH 私钥登录

步骤：

1. 创建主机配置。
2. 选择 private key。
3. 选择私钥文件。
4. 如需要，输入 passphrase。
5. 点击连接。
6. 执行 `hostname`。

预期：

- 连接成功。
- 终端输出服务器 hostname。
- passphrase 不出现在日志中。

### 4.3 认证失败

步骤：

1. 创建主机配置。
2. 输入错误密码。
3. 点击连接。

预期：

- 连接失败。
- 返回 `SSH_AUTH_FAILED`。
- UI 显示认证失败。
- 不泄漏密码。

### 4.4 连接超时

步骤：

1. 创建不可达地址。
2. 点击连接。

预期：

- 超时后返回 `SSH_CONNECT_TIMEOUT` 或 `SSH_NETWORK_UNREACHABLE`。
- UI 不一直转圈。
- 可以重试或编辑配置。

### 4.5 终端 resize

步骤：

1. 连接服务器。
2. 打开终端。
3. 调整窗口大小。
4. 执行 `stty size`。

预期：

- 远端 PTY 尺寸与前端终端尺寸匹配或接近。
- resize 后终端不乱行。

### 4.6 多标签终端

步骤：

1. 连接同一主机。
2. 打开两个终端标签。
3. 分别执行不同命令。
4. 切换标签。

预期：

- 两个标签互不串流。
- 关闭一个标签不影响另一个标签。

### 4.7 SFTP 浏览

步骤：

1. 连接服务器。
2. 打开 SFTP 面板。
3. 浏览 home 目录。
4. 进入子目录。
5. 返回上级目录。

预期：

- 文件列表正确。
- 文件和目录图标区分。
- 路径变化正确。
- 权限不足时显示错误。

### 4.8 SFTP 上传下载

步骤：

1. 上传一个小文件。
2. 上传一个较大文件。
3. 下载上传后的文件。
4. 对比文件大小。

预期：

- 传输队列显示进度。
- 完成后状态为 completed。
- 文件大小一致。
- 失败时可重试。

### 4.9 SFTP 文件操作

步骤：

1. 新建目录。
2. 重命名目录。
3. 上传文件。
4. 重命名文件。
5. 删除文件。
6. 删除目录。

预期：

- 所有操作成功。
- 文件列表刷新。
- 删除操作有确认。

### 4.10 本地端口转发

步骤：

1. 创建 local forwarding 规则。
2. 本地监听 `127.0.0.1:18080`。
3. 远程目标指向测试服务端口。
4. 启动规则。
5. 从本机访问 `127.0.0.1:18080`。

预期：

- 转发启动成功。
- 访问可通。
- 停止规则后访问失败。

### 4.11 远程端口转发

步骤：

1. 确认 SSH 服务端允许 remote forwarding。
2. 创建 remote forwarding 规则。
3. 启动规则。
4. 从远端或指定位置访问远程监听端口。

预期：

- 转发启动成功，或明确提示服务端不允许。
- 停止后释放端口。

### 4.12 动态 SOCKS 转发

步骤：

1. 创建 dynamic forwarding 规则。
2. 本地监听 SOCKS 端口。
3. 配置本机请求使用 SOCKS。
4. 访问远端可达目标。

预期：

- SOCKS 转发可用。
- 停止后端口释放。

### 4.13 监控采集

步骤：

1. 连接 Linux 服务器。
2. 打开监控面板。
3. 等待至少三次刷新。

预期：

- 展示 CPU、内存、磁盘、网络、uptime。
- 刷新不影响终端输入。
- 命令失败时面板显示不可用原因。

### 4.14 进程列表

步骤：

1. 打开进程面板。
2. 刷新进程。
3. 搜索进程名。
4. 按 CPU 排序。

预期：

- 列表正常展示。
- 搜索和排序生效。
- kill 操作前有确认。

### 4.15 Vault

步骤：

1. 创建带保存密码的主机。
2. 退出应用。
3. 重新打开应用。
4. 使用已保存凭据连接。
5. 检查配置文件和日志。

预期：

- 可以读取保存凭据。
- 文件中没有明文密码。
- 日志中没有明文密码。

## 5. UI 验收

### 5.1 启动体验

预期：

- 启动后直接进入工作台。
- 不出现营销首页。
- 后端初始化时 UI 有明确状态。
- sidecar 未就绪时不崩溃。

### 5.2 工作台布局

预期：

- 左侧主机入口清晰。
- 中央终端区域突出。
- 文件、转发、监控、进程入口可快速找到。
- 多标签不挤压主操作区。
- 面板 resize 不破坏布局。

### 5.3 视觉风格

预期：

- 深色优先。
- 色彩克制。
- 分割线清晰。
- hover 和 selected 状态明确。
- 字体清晰。
- 图标精致。
- 不出现大面积渐变。
- 不出现营销式卡片堆叠首页。

### 5.4 响应状态

必须覆盖：

- 连接中。
- 已连接。
- 断开。
- 认证失败。
- 文件传输中。
- 文件传输失败。
- 端口转发运行中。
- 端口转发失败。
- Vault 锁定。
- Vault 解锁失败。

## 6. 安全验收

第一阶段安全验收：

- 密码不明文写入 profile store。
- passphrase 不明文写入 profile store。
- 可选私钥内容不明文写入 profile store。
- 日志不包含敏感凭据。
- Vault 文件包含 schema version。
- Vault 文件损坏时不会覆盖原文件。
- 主密码错误不能解锁 Vault。

第一阶段不验收：

- Keychain。
- Credential Manager。
- known_hosts。
- 首次指纹确认。
- 指纹变更阻断。

这些能力进入第二阶段验收。

## 7. 回归测试清单

每次发布前至少回归：

- 应用启动。
- Java sidecar 启动。
- 主机列表加载。
- 密码 SSH 登录。
- 私钥 SSH 登录。
- 终端输入输出。
- 终端 resize。
- 多标签切换。
- SFTP 列目录。
- SFTP 上传。
- SFTP 下载。
- 本地端口转发。
- 动态 SOCKS 转发。
- 监控刷新。
- Vault 解锁。
- 应用退出清理连接。
