## 本阶段实现

### Electron Builder

新增配置：

```text
apps/desktop/electron-builder.yml
```

关键点：

- `appId`：`com.termira.desktop`
- `productName`：`Termira`
- `electronVersion`：`41.3.0`
- `electronDist`：使用本地 `node_modules/electron/dist`，减少打包时网络依赖。
- `icon`：`apps/desktop/build/icon.icns`
- `extraResources`：将 `apps/backend-java/target/termira-backend.jar` 复制到 `backend-java/termira-backend.jar`。

### macOS DMG

Electron Builder 负责 `.app` 和 `.zip`，DMG 由本地脚本调用 `hdiutil` 生成：

```text
apps/desktop/scripts/create-mac-dmg.mjs
```

原因：

- Electron Builder 默认 DMG 定制流程需要下载额外 dmg-builder 辅助包。
- 当前阶段优先保证本机可重复打包，使用 macOS 自带 `hdiutil` 生成可挂载 DMG。

### Renderer file 协议修复

Vite 新增：

```ts
base: "./"
```

原因：

- 打包版通过 `file://.../app.asar/dist/index.html` 加载 renderer。
- 原先 `/assets/...` 绝对路径会导致打包版黑屏。
- 改成相对路径后，packaged app UI 正常加载。

### Java sidecar fat jar 修复

Maven shade 排除依赖签名文件：

```xml
META-INF/*.SF
META-INF/*.DSA
META-INF/*.RSA
```

原因：

- 依赖签名文件进入 shaded jar 后，`java -jar termira-backend.jar` 会报 `Invalid signature file digest`。
- 排除签名文件后，打包内 sidecar jar 可正常启动。

### 私钥登录修复

修复 `privateKeyPath` 无 passphrase 时调用 SSHJ `loadKeys(path, String)` 传入 `null` 导致 `NullPointerException` 的问题。

当前实现改为：

```java
client.loadKeys(keyPath.toString(), passwordFinder(spec.passphrase()))
```

结果：

- 无 passphrase 的 Ed25519 私钥路径登录通过真实服务器 E2E。

## 人工验收步骤

1. 打开：

   ```text
   apps/desktop/release/Termira-0.1.0-arm64.dmg
   ```

2. 将 `Termira.app` 拖入 Applications。
3. 首次打开时，如果 macOS 提示未签名应用，需要在系统安全设置中允许打开。
4. 启动后确认：
   - 首屏为 Termira 工作台。
   - 左侧导航和主机列表显示正常。
   - 设置页能看到后端状态。
5. 创建测试主机：
   - host：`<host>`
   - user：`<user>`
   - auth：password 或 private key
6. 点击连接并打开终端。
7. 执行：

   ```bash
   whoami
   hostname
   pwd
   ```

8. 验收 SFTP：
   - 列目录。
   - 上传小文件。
   - 下载小文件。
   - 删除测试文件。
9. 验收端口转发：
   - 本地转发。
   - 动态 SOCKS。
   - 远程转发。
10. 验收监控：
    - 资源刷新。
    - 进程列表。
11. 退出应用，确认 Java sidecar 进程随应用退出。

## 已知问题

- macOS 包当前未签名、未 notarize。
- macOS 包当前仅在 arm64 环境验证。
- 打包版仍依赖系统可用 `java` 命令，后续建议引入 jlink runtime。
- Electron Builder 安装阶段在 Node 20 下会提示 `@electron/rebuild`/`node-abi` 期望 Node 22.12+，当前因 `npmRebuild=false` 且无 native addon，不影响本阶段打包。
- Vite chunk 大小超过 500 KB 有 warning，第一阶段不影响功能，后续可按路由和大型组件拆包。
